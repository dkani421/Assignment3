Project 2: Part 1

Project Description and Requirements

For this project, I developed a web application using MySQL and PHP to provide an online bookmarking service to users on the Internet. The site allows for the saving and editing of bookmarks and also lets the user see the top 10 bookmarks saved by all.

Analysis and Design

To accomplish the requirements of Part 1, I followed the following analysis and design approach:

  • Requirement analysis: I thoroughly analyzed the project requirements to identify the key features and functionalities of the web application.
  • Interface design: I designed a thoughtful interface with menus and navigation buttons to enhance user experience and ensure ease of use.
  • Database design: I created a database structure to store and manage the user's bookmark data efficiently.
  • Validation design: I implemented JavaScript validation to ensure that the URLs provided by the user are not only correct but also active.

Implementation Documentation

The implementation of Part 1 involved the following steps:

  1. Front-end development: I used HTML and CSS to develop the user interface, including the welcome message and bookmark lists. The interface was designed to be visually appealing and intuitive for users.
  2. Back-end development: I utilized PHP to handle user authentication, perform database operations, and validate user inputs. The PHP code was structured and modular, adhering to best practices.
  3. Database integration: I employed MySQL to store and manage the user's bookmark data. The database tables were designed to efficiently store and retrieve the required information.

User Guide

The following user guide provides instructions on how to use the web application developed for Part 1:

  1. Access the web application using the provided link.
  2. Upon landing on the homepage, you will be greeted with a welcome message and a list of the ten most popular websites bookmarked by users.
  3. To sign in, click on the appropriate link/button and enter your credentials. For now you can use the username, password combo admin, admin.
  4. After signing in, you will be able to view your list of bookmarks.
  5. To add a new website, click on the "Add Bookmark" button and provide the necessary details.
  6. You can edit or delete existing bookmarks by selecting the appropriate options.
  7. When adding or editing bookmarks, make sure to provide valid and active URLs to ensure proper functionality.

Project 2: Part 2

Project Description and Requirements

In Part 2 of the assignment, I developed a small-scale online learning management system (LMS) that can be used to deliver online courses to learners. The requirements included the following aspects:

  • Designing a subject matter expert (SME)-friendly language (Educational Markup Language - EML) for marking up educational materials.

    EML formatting: teacherInput.eml

  • Storing the course contents on the web server using a database for efficient management and retrieval.

    Database creation: database_creation.sql

  • Implementing a user management module for learner registration and login.
  • Allowing learners to view and select lessons within the system.
  • Developing a parser in PHP to translate EML content into HTML for storage and proper rendering.

Analysis and Design

To successfully meet the requirements of Part 2, I followed an analysis and design approach:

  • System analysis and design: I conducted a comprehensive analysis of the requirements, considering the data flow, functional modules of the learning management system, and the relationships among these modules. I also designed the system layout within the browser's window to ensure a seamless user experience.
  • Educational Markup Language (EML): I designed an EML that is user-friendly for subject matter experts (SMEs) to create educational materials. The EML included tags and their intended purposes, along with examples to aid in understanding and usage.
  • Database design: I designed the database structure to efficiently store information about courses, units, lessons, quizzes, learners, and other learning objects such as images, audio, and video files. The design included appropriate fields to identify the course and unit to which each lesson or quiz belongs.
  • Parser design: I implemented a PHP parser that retrieves the EML content from the database and translates it into HTML documents for proper rendering on a web browser.The parser was designed using PHP's regular expression functions for pattern matching and string replacement, ensuring accurate conversion from EML to HTML.

Implementation Documentation

The implementation of Part 2 involved the following steps:

  1. Front-end development: I utilized HTML and CSS to create the user interface for the learning management system. The interface was designed to be visually appealing and intuitive, providing a seamless learning experience.
  2. Back-end development: PHP was used to implement the user management module, allowing learners to register and obtain accounts within the system. Additionally, the backend was responsible for retrieving and delivering the course contents to the web browser, utilizing the parser to translate EML content into HTML for proper rendering.
  3. Database integration: MySQL was used to store and manage various aspects of the learning management system, including course information, unit details, lesson and quiz content, learner data, and other associated learning objects such as images, audio, and video files. SQL scripts were provided to create the necessary tables and PHP files were developed to access the database for seamless integration.

User Guide

To effectively use the online learning management system developed for Part 2, please follow the instructions outlined below:

  1. Access the web application using the provided link.
  2. If you are a new user, register for an account by providing the necessary details.
  3. Once registered, log in to the system using your credentials.
  4. Upon successful login, you will be presented with a list of available lessons and quizzes.
  5. Select a lesson to start or continue your learning journey.
  6. The lesson content, written in EML, will be dynamically parsed into HTML and presented on your web browser for a seamless learning experience.
  7. Complete quizzes associated with each lesson to assess your understanding.
  8. Explore other functionalities provided by the learning management system, such as course selection, grade services, and more.
