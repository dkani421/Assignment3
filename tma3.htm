Project Description and Requirements

Part 1 of this project involves creating a web application using ASP.NET in C# that utilizes a persistent cookie to track the number of times a client computer has visited the page. The application is also required to display the IP address of the client computer and the time zone of the client's location.

Implementation Documentation

The implementation of Part 1 includes the following steps:

  • ASP.NET Application: Developed a web application using ASP.NET and C# to fulfill the project requirements.
  • Persistent Cookie: Implemented a persistent cookie to keep track of the number of visits by the client computer.
  • IP Address Retrieval: Utilized ASP.NET to retrieve and display the IP address of the client computer.
  • Time Zone Display: Incorporated functionality to show the time zone of the client computer's location.

User Guide

Project 3: Part 3

Project Description and Requirements

Part 3 of this project marks the development of an online computer store where customers can customize computers and place orders. The application is designed using ASP.NET in C# as the server-side language.

  • Research and Analysis: Begin by studying the design and functionalities of popular online stores like Bestbuy.CA and Amazon.CA.
  • Design and Interface: Based on my research, I analyzed the requirements of the online store, Developed a design that features a realistic web browser-based interface. The design includes a store name, logo, navigational menus, buttons, and essential pages such as a welcome page, contact page, and feedback form.
  • Computer Listings: The online store showcases a list of computers, along with lists of various components (RAM, hard drive, CPU, display, OS, soundcard, etc.) that customers can review.
  • Customization: Customers can select a computer with its default configuration and pricing. The application will allow customers to customize the computer by changing components. The replacement will occur from a list of compatible components.
  • Dynamic Price Update: Whenever a customer makes a change to the computer's configuration, the total price should be dynamically recalculated and displayed within the online store, integrated into the page layout (not as a pop-up).
  • Order Summary: After a customer confirms their selections, such as clicking "Buy" or "Add to Cart," a summary of the chosen components, customized configuration, and the associated pricing will be displayed in the browser window.
  • Temporary Storage: Orders placed by customers use Cookies to store and track customer selections temporarily.

Implementation Documentation

The implementation of Part 3 includes the following steps:

  • Design Approach: Designed an intuitive and user-friendly web interface that emulates a real online store.
  • Component Lists: Incorporated the list of computers and component options into the HTML or program files.
  • Customization Logic: Develop the logic for customer customization, ensuring compatibility and accurate pricing adjustments.
  • Order Summary Display: Design and program the display of the order summary, featuring selected components and pricing.
  • Cookies Usage: Utilized Cookies to temporarily store customer selections and customizations.

User Guide

The following user guide provides instructions on using the web application developed for Part 3:

  1. Access the web application using the provided link.
  2. Navigate through the online store's interface, exploring computers and component options.
  3. Select a computer with its default configuration or customize it by replacing components from the compatible list.
  4. Observe the dynamic price updates as you make changes to the computer's configuration.
  5. Upon confirmation of selections, review the order summary displaying your chosen components and pricing.

Project 3: Part 4

Project Description and Requirements

Part 4 of this project extends the work accomplished in Part 3, focusing on the completion of a fully functional online computer store. The following tasks are included in this part of the assignment:

  • Customer Management Module: Integrate a customer management module into the online computer store. This module should encompass new customer registration, customer authentication, and password recovery functionalities.
  • Order Management Module: Develop an order management module that enables the saving of orders for registered customers. Registered customers should be able to manage their orders, including viewing lists and details of each item in the order, deleting or modifying selected orders.
  • Database Design and Implementation: Design and implement a back-end database that supports the online computer store's functionality. The database should fulfill the following tasks:
    • Store Details: Store details of computers and components, replacing any hardcoded data in program files.
    • Customer Management: Utilize the database for all customer-related data, including registration details.
    • Order Management: Employ the database for all aspects of order management. Upon order submission, save order details including pricing in the database.

Implementation Documentation

The implementation of Part 4 comprises the following steps:

  • Customer Module Integration: Incorporate a new customer management module, enabling new customer registration, authentication, and password recovery.
  • Order Management: Develop the order management module, allowing registered customers to view, modify, or delete their orders.
  • Database Creation: Design the database schema to accommodate computers, components, customers, and orders.
  • Data Integration: Replace hardcoded data in program files with data fetched from the database.
  • Database Usage: Implement the database for customer management and order management, including order details and pricing storage.

User Guide
